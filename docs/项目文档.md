# PasswordManager - 密码管理器

## 项目概述

PasswordManager 是一个基于 iOS 平台的密码管理应用，主要用于安全存储和管理加密文件与用户凭证的映射关系。项目采用 App Group 共享容器机制，实现主应用与 Share Extension 之间的数据共享，并通过文件尾部元数据标记实现跨应用文件追踪。

## 核心功能

### 当前实现（第二阶段）
- **4大业务通路**：统一查密、WPS纯看未改、野生文件登记、WPS正常写回
- **文件尾部元数据**：通过 WPPM 标记在文件尾部存储 UID 和密码信息，支持多标记顺序搜索
- **后台密码撞击**：从数据库取前5条活跃记录密码队列进行自动解密尝试
- **SQLite WAL 模式**：多进程并发硬化控制，2秒 Busy Timeout 避让重试锁
- **主应用保洁总线**：僵尸索引自动清洗 + LRU 磁盘空间限额控制
- **状态D惰性滑动认领画布**：ScrollView + LazyVStack 分页加载
- **主应用资产保险箱界面**：移除导航栏标题，新增顶部搜索框（支持模糊搜索），文件列表展示（按最后访问时间排序）、详情查看（文件名、大小、存储位置、UID）、密码复制（从文件尾部读取）、文件删除（物理删除 + 数据库记录删除）
- **重名检测与重命名**：登记新文件时检测保险箱是否有同名文件，若有则弹出对话框要求用户输入新文件名（默认填充原文件名）
- **关联覆盖密码验证**：点击"关联并覆盖本地已有文件"按钮时，需要先验证用户输入的密码，验证通过后才显示资产选择列表（状态D）
- **UID优先从文件尾部读取**：关联覆盖时优先从被覆盖文件尾部读取UID，若不存在则使用数据库中的UID
- **文件预览自动复制密码**：点击文件条目时，异步从文件尾部读取密码并复制到剪贴板，显示非阻塞式Toast提示。将文件复制到系统临时目录后，通过 `UIApplication.shared.open()` 直接唤起第三方应用（如 WPS）打开文件。用户在第三方应用的密码输入框中长按粘贴即可使用。剪贴板30秒后自动清理，临时文件5秒后自动清理，确保安全性
- **Office密码验证方式精简**：OfficeCryptoVerifier仅保留Standard Encryption和Agile Encryption 4.4两种有效验证方式，清理冗余的二进制解析和未使用的辅助方法
- **状态D返回导航修复**：从蓝色画布（A/B）进入状态D后点击"返回上一步"，正确回到原蓝色画布，而非错误地回到橙色画布
- **蓝色画布逻辑优化**：蓝色画布A和B统一"另存为新文件"和"覆盖其他文件"按钮，添加密码验证逻辑，匹配提示区分文件/记录
- **另存为新文件Bug修复**：使用文件自身UID而非数据库匹配记录的UID，避免覆盖原有记录
- **主页搜索功能优化**：新增搜索索引（SearchIndex），在主页打开或刷新时预构建倒排索引，支持字符级n-gram分词的模糊搜索；搜索结果按前缀匹配、完全匹配、字符匹配度综合评分排序
- **文件打开方式优化**：优先使用 `UIApplication.shared.open()` 唤起外部应用（如WPS），失败时自动降级到 `UIDocumentInteractionController` 系统预览；创建 `DocumentInteractionManager` 类管理文档交互控制器的生命周期，解决预览控制器被过早释放的问题

## 核心模块说明

### 1. AppGroupDBManager

负责管理 App Group 共享容器中的 SQLite 数据库，提供文件映射的 CRUD 操作。

**数据库表结构:**

| 字段 | 类型 | 说明 |
|------|------|------|
| uid | TEXT | 文件唯一加密基因（主键） |
| file_name | TEXT | 标准化小写文件名 |
| password_hash | TEXT | 密码密文句柄/解密盐值 |
| last_access_time | INTEGER | 最后访问时间戳（用于LRU排序） |
| file_size | INTEGER | 物理文件大小（辅助特征判定） |
| is_local_vault | INTEGER | 0=外部纯记账，1=落盘保险箱 |

**索引:**
- `idx_filename_size`: file_name + file_size（辅助模糊匹配）
- `idx_access_time`: last_access_time（LRU排序）

**主要方法:**

| 方法 | 功能 |
|------|------|
| saveFileMapping | 保存文件映射记录 |
| upsertRecord | 插入或更新记录（INSERT OR REPLACE） |
| updateAccessTime | 更新访问时间 |
| updateVaultStatus | 更新保险箱状态 |
| queryRecordByUID | 按UID查询完整记录 |
| queryRecordsByFileName | 按文件名模糊查询 |
| queryTopActiveRecords | 获取前N条活跃记录（密码撞击） |
| queryAllLocalVaultRecords | 获取所有本地保险箱记录 |
| deleteRecord | 删除记录 |
| fetchAllLog | 获取所有记录日志 |

### 2. ZipExtraFieldManager

负责在文件尾部读写 WPPM 格式的元数据标记。

**元数据格式:**
```
WPPM + Version(2) + Type(1) + Length(4) + Content + CRC32(4)
```

**多标记存储规则:**
- 一个文件尾部可存储多个WPPM标记，按顺序依次写入：`[UID标记][PASSWORD标记][KEY_VERSION标记]`
- 读取时从文件尾部向前搜索，找到最新的对应类型标记

**主要方法:**

| 方法 | 功能 |
|------|------|
| readUid | 读取文件尾部的UID标记 |
| readPassword | 读取文件尾部的密码标记 |
| readKeyVersion | 读取文件尾部的密钥版本标记 |
| writeMetadata | 写入UID、密码和密钥版本元数据 |
| removeOldWppmMarkers | 清除旧的WPPM标记 |

### 3. OfficeCryptoVerifier

负责验证 Office 文件密码的正确性，支持 OLE2 复合文档格式的 Standard Encryption (version 4.2) 验证。

**核心验证流程:**
1. **文件格式识别**: 识别 ZIP 格式和 OLE2 复合文档格式
2. **OLE2 解析**: 实现完整的 OLE2 复合文档解析器，支持 FAT/DIFAT 读取、目录项遍历、流读取
3. **EncryptionInfo 解析**: 解析 Office 加密元数据（salt、encryptedVerifier、encryptedVerifierHash）
4. **密钥派生**: 按照 ECMA-376 标准，SHA-1 迭代 50000 次 + fillAndXor 操作生成 AES 密钥
5. **密码验证**: AES-ECB 解密 verifier 和 verifierHash，比对验证

**主要方法:**

| 方法 | 功能 |
|------|------|
| verifyPasswordAsync | 异步验证文件密码 |

**新增内部类:**

| 类 | 功能 |
|------|------|
| OLE2Parser | OLE2 复合文档解析器，支持 FAT/DIFAT/目录项/流读取 |

### 4. ShareExtensionView

分享扩展主视图，实现4大业务通路和5种画布状态。

**宿主类型检测机制：**

通过检测文件来源路径中是否包含 `wps_share_temp_dir` 字符串来判定宿主类型：

| 宿主类型 | 判定条件 | 说明 |
|----------|----------|------|
| WPS | 路径包含 `wps_share_temp_dir` | WPS 分享临时目录特征 |
| 外部 | 路径不包含该特征 | 微信、QQ、文件App等 |

**画布颜色体系:**

| 颜色 | 场景 | 触发通路 | 动作按钮 |
|------|------|----------|----------|
| 🟢 绿色 | 统一查密画布 | 通路1 | 复制密码并退场（直接退出，无倒计时）、取消 |
| 🟠 橙色 | 登记新文件画布 | 通路3 | 登记为本地新文件（含重名检测）、覆盖其他文件（需验证密码）、取消 |
| 🔵 蓝色A | 精准同步确认画布 | 通路2/4撞击成功 | 确定同步覆盖、另存为新文件、覆盖其他文件、取消（匹配提示区分文件/记录） |
| 🔵 蓝色B | 强制密码输入画布 | 通路4撞击失败/通路2数据库无记录 | 另存为新文件、覆盖其他文件、取消 |
| ⚪ 状态D | 惰性滑动认领画布 | 手动关联触发 | 列表行卡片、返回上一步 |

**4大通路详细说明:**

**通路1：统一查看密码线**
- 适用场景：外部微信/QQ收到带元数据文件，或保险箱目录直接分享
- 用户心智："临时查个密码，自己手动打开"
- 核心动作：元数据提取（优先从文件尾部提取密码，不再查询数据库）→ 统一查密画布 → 复制密码并退场 → REPLACE INTO记账 → 自动闪退
- 密码提取优先级：① 文件尾部元数据密码 ② 数据库记录（已移除）③ 默认密码（已移除）
- 无密码场景：按钮文案变为"当前文档无密码"

**通路2：WPS纯看未改写回线**
- 适用场景：WPS内打开带元数据文件，纯阅读未修改后分享回插件
- 物理特征：WPS未触发重写，尾部元数据保留
- 核心动作：闪电免撞击 → 提取UID → 数据库反查 → 直通同步确认画布

**通路3：野生资产首次实名登记线**
- 适用场景：外部传入无元数据的加密文件
- 用户心智："收纳野生资产到企业保险箱"
- 核心动作：乱码纠偏 → 黄色画布强制密码输入 → OfficeCryptoVerifier验证 → 重名检测 → 生成新UID → 落盘保险箱
- **重名检测机制**：登记新文件时，先检查保险箱目录是否已有同名文件，如果有则弹出对话框要求用户输入新文件名（默认填充原文件名）
- **关联覆盖密码验证**：点击"关联并覆盖本地已有文件"按钮时，需要先验证用户输入的密码，验证通过后才显示资产选择列表（状态D）

**通路4：WPS正常编辑写回线**
- 适用场景：WPS编辑合规文件后分享回插件
- 物理特征：WPS重编译，尾部元数据被抹空
- 核心动作：后台密码撞击（前5条活跃密码）→ 成功直通蓝色A → 失败降级蓝色B

**关联覆盖逻辑（状态D）:**
- 当用户选择用新文件A覆盖老文件B时：
  - **UID获取**：优先从B文件尾部读取UID，若不存在则使用数据库中的UID
  - **密码使用**：使用用户在密码输入框中验证通过的密码
  - **文件名**：保留B文件的文件名（继承被覆盖文件的身份）
  - **执行流程**：将B的UID + 用户密码写入A文件尾部 → 将A移动覆盖到B的位置 → 更新数据库记录

**重名检测流程:**
1. 用户点击"登记为本地新文件"
2. 调用 `fileNameExistsInVault(fileName:)` 检查保险箱目录是否存在同名文件
3. 若存在重名，弹出重命名对话框（`showRenameDialog = true`），默认填充原文件名
4. 用户输入新文件名后点击"确定"，调用 `confirmRename()` 执行注册
5. 若新文件名仍为空，提示错误并阻止继续

**关联覆盖密码验证流程:**
1. 用户在黄色画布输入密码后点击"覆盖其他文件"
2. 调用 `verifyAndShowAssetSelection()` 触发密码验证
3. 使用 `OfficeCryptoVerifier` 验证密码是否正确
4. 验证通过后调用 `showAssetSelection()` 加载资产列表并切换到状态D
5. 验证失败则显示错误提示，阻止进入资产选择列表

**蓝色画布优化说明:**
- **蓝色画布A**：匹配提示根据 `is_local_vault` 字段区分显示（"匹配到文件：xxx"或"匹配到记录：xxx"），所有按钮操作前先验证密码，失败则弹窗输入；另存为新文件使用文件自身UID（无则新生成）
- **蓝色画布B**：统一按钮文案（另存为新文件、覆盖其他文件），添加密码验证逻辑，移除旧按钮（确定同步覆盖、确认登记并保留身份）

### 5. PasswordManagerApp

主应用入口，包含后台保洁总线和资产保险箱界面。

**保洁总线:**

| 功能 | 触发时机 | 执行逻辑 |
|------|----------|----------|
| 僵尸索引清洗 | 应用进入前台 | 扫描保险箱目录，删除物理文件不存在的数据库记录 |
| LRU磁盘清理 | 应用进入前台 | 超过1GB阈值时，按last_access_time升序删除最冷门文件 |
| 过期临时文件清理 | 应用启动/进入前台 | 清理24小时前的临时文件 |

**资产保险箱界面:**

主应用的资产保险箱界面基于 SwiftUI 实现，采用 `NavigationStack` + `List` 结构，提供完整的资产管理功能。

**界面组件:**
- **AssetRow**：文件列表行组件，展示文件图标、文件名、大小、最后访问时间和保险箱状态标识
- **AssetDetailView**：文件详情页，展示文件完整信息和密码复制功能
- **FileIconView**：文件类型图标组件，根据扩展名显示不同图标（.docx蓝色、.xlsx绿色、.pptx橙色、.pdf红色）
- **SearchBar**：顶部搜索框组件，支持实时模糊搜索文件名

**功能说明:**
- **文件列表展示**：按 `last_access_time` 降序排列，最新访问的文件排在最前；支持下拉刷新和手动刷新按钮
- **模糊搜索功能**：顶部搜索框支持实时模糊搜索，使用倒排索引实现高效匹配；搜索时忽略大小写，支持中文友好匹配；搜索结果按前缀匹配、完全匹配、字符匹配度综合评分排序
- **文件预览与自动复制密码**：点击文件条目时，异步从文件尾部读取密码并复制到剪贴板，显示非阻塞式Toast提示。将文件复制到系统临时目录后，优先通过 `UIApplication.shared.open()` 唤起第三方应用（如 WPS）打开；若唤起失败则降级到 `UIDocumentInteractionController` 系统预览。用户在第三方应用的密码输入框中长按粘贴即可使用。剪贴板30秒后自动清理，临时文件5秒后自动清理，确保安全性
- **文件详情查看**：点击锁图标进入详情页，展示文件名、文件大小、最后访问时间、存储位置、UID等信息
- **密码复制功能**：从文件尾部读取 WPPM 密码标记，点击"复制密码"按钮将密码复制到剪贴板，2秒后按钮状态恢复为"复制密码"
- **文件删除功能**：支持左滑删除，同时执行物理文件删除和数据库记录删除

**关键实现:**
- 列表数据通过 `queryAllLocalVaultRecords()` 从数据库获取，仅显示 `is_local_vault = 1` 的记录
- 搜索索引（SearchIndex）：在主页打开或刷新时预构建倒排索引，采用字符级n-gram分词（单字和双字组合）；搜索时直接查索引，无需遍历列表
- 密码读取通过 `ZipExtraFieldManager.shared.readPassword(from:)` 从文件尾部读取
- 文件打开流程：将文件从保险箱复制到系统临时目录 → 优先 `UIApplication.shared.open()` 唤起外部应用 → 失败时降级 `UIDocumentInteractionController` 系统预览 → 预览关闭后自动清理临时文件 → 更新数据库 `last_access_time`
- `DocumentInteractionManager`：管理文档交互控制器的生命周期，确保控制器和delegate不被过早释放
- Toast提示采用非阻塞式覆盖层设计，不影响打开流程
- 文件删除时先删除物理文件，再删除数据库记录，确保数据一致性

## 开发注意事项

1. **数据库并发**: 使用 WAL 模式和 `sqlite3_busy_timeout(2000)` 处理并发写入冲突
2. **文件名规范化**: 所有文件名存储前进行小写转换和空白字符清理
3. **线程安全**: 后台密码撞击在 `Task.detached` 中执行，避免阻塞主线程
4. **文件协调**: 使用 `NSFileCoordinator` 处理跨卷文件操作
5. **备份排除**: 保险箱文件设置 `isExcludedFromBackup = true`
6. **剪贴板安全**: 密码复制到剪贴板后由用户自行管理，点击"复制密码并退场"后直接退出扩展
7. **临时文件管理**: 文件打开时需要复制到系统临时目录（App Group 容器路径对第三方应用受限），打开成功后5秒自动清理
